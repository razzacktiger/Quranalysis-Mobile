# Issue Resolution Patterns

**Purpose:** Document issues encountered and their solutions for future sessions.

## How to Use This File

**When you encounter and resolve an issue:**
1. Add it under the appropriate category file
2. Include: Symptom, Cause, Fix, Prevention
3. Keep entries concise

**When starting a new session:**
1. Review relevant category based on task type
2. Check if current issue matches a known pattern

## Adding New Learnings

Use this template:

```markdown
### [Brief Description]
**Symptom:** What the user/agent saw
**Cause:** Root cause analysis
**Fix:** How it was fixed
**Prevention:** How to avoid in future tasks
```
